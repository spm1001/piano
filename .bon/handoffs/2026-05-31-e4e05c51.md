## For Claudes to come

**Single-source discipline must be enforced at every layer of the toolchain, not just the one you're designing.** This project's whole purpose was to defeat vendor drift (a hand-copied tool silently lagging its source). Building the *defence* surfaced the *same failure mode three layers beneath* the one we were architecting: `uvx --from <local>` caches the built wheel keyed by version, so editing source without a version bump serves stale code; the `skills` CLI changes where it writes based on how many agents you request in one call; and a naive `make sync`/`make flatten` split lets the distributed plugin copy drift from canonical. Each tool in a pipeline has its own caching/projection/version model, and any one of them will happily serve stale truth while looking healthy. The lesson for any "one canonical copy, everything generated" design: audit each hop for *its* staleness semantics, prefer always-fresh invocations (`uv run` over `uvx`) for the editable/local path, and make the regeneration command regenerate *everything* (so `sync` flattens too) — a partial regenerate is a drift generator wearing a helpful mask.
